Netlify に GitHub リポジトリをつないで公開する手順

対象リポジトリ：private-portal（index.html はリポジトリ直下）— Publish directory は /

GitHub 連携 ビルド不要（HTMLのみ） 無料で公開
1

Netlify でプロジェクト作成を開始

Netlify ダッシュボード右上の Add new project → Import an existing project をクリック。

⚠️
見つからない場合：左ナビ Projects ページ右上でも同じボタンがあります。
2

GitHub を接続してリポジトリを選択

  • GitHub を選択 → GitHub の認証画面で Authorize Netlify を許可。
  • リポジトリ一覧から private-portal を選択。
  • ブランチは main を指定。
3

ビルド設定（HTMLのみなので超シンプル）

  • Build command：空欄（入力しない）
  • Publish directory：/（スラッシュ）
Publish directory: /
✅
理由：index.html がリポジトリ直下にあるため、公開するディレクトリはルート（/）でOKです。
4

Deploy site（公開）

Deploy site を押すと数十秒ほどでデプロイが完了し、https://〇〇〇.netlify.app のような公開URLが払い出されます。

💡
公開URLはプロジェクトの Project overview → サイト名の下に表示されます。
5

動作確認 & 以降の更新

  • 払い出された URL にアクセスし、ページ表示を確認。
  • 以降は GitHub の main ブランチに push すると Netlify が自動で再デプロイします。
6

（任意）サイト名や独自ドメイン

  • サイト名変更：Site settings → Change site name
  • 独自ドメイン：Domain management → Add custom domain
7

トラブルシューティング

  • 404 や空の画面になる：Publish directory が / か再確認。
    もし index.html がサブフォルダなら、そのフォルダ名（例：public）を指定。
  • 画像/JS/CSS が表示されない：パスが絶対参照（例：/img/logo.png）になっていないか。相対パス推奨。
  • ビルドエラー：HTMLのみの場合は Build command を空欄にする。
📌
GitHub 側でファイル名の大文字小文字が変わると、Linux環境では別ファイル扱いになります。拡張子や綴りの一致を確認してください。
8

とにかく最短で公開だけしたいとき

Netlify Drop（ドラッグ＆ドロップ公開）を使う方法もあります：

  • Netlify Drop を開く
  • index.html を含むフォルダをまとめてドラッグ＆ドロップ
  • すぐに一時的なURLが払い出されます
⚠️
単一の .html ファイルだけをドロップしても公開できません。必ず index.html を含むフォルダを落としてください。
